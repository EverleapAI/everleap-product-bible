# Today

## Purpose

Today is the heartbeat of Everleap.

It is the page most users should visit every day.

Unlike the rest of Everleap, which focuses on long-term understanding, Today focuses on helping the user make one meaningful step forward today.

It bridges self-understanding and real life.

Understanding without action has limited value.

Today exists to encourage action.

---

# The Question Today Answers

Every page in Everleap answers one question.

Today's question is:

> **"Given everything we currently know, what would be most helpful today?"**

Not this week.

Not next year.

Today.

---

# The Philosophy

People rarely change because of one profound realization.

They change because of many small moments.

One observation.

One conversation.

One experiment.

One success.

One mistake.

One reflection.

Today should create those moments.

---

# The User Experience

When opening Today, the user should immediately feel:

"This was written for me."

Not because it flatters them.

Not because it predicts them.

Because it reflects what the product currently understands about where they are in their journey.

Today's guidance should always feel:

- personal
- timely
- thoughtful
- encouraging
- achievable

---

# What Today Contains

The Today page exists to answer four questions.

## 1. What seems most important right now?

This becomes today's primary guidance.

It should be generated from the strongest available evidence.

Not from generic advice.

---

## 2. Why might this matter?

The product should briefly explain why today's idea may be meaningful.

Users should understand the reasoning without seeing technical details.

---

## 3. What is one small thing to try?

Every Today experience should encourage one realistic action.

Actions should feel achievable.

Never overwhelming.

---

## 4. What can we learn from it?

Every action becomes new evidence.

Growth comes from observation.

Not perfection.

---

# Personalization

Today's guidance should evolve continuously.

It should consider:

- onboarding
- Story
- completed actions
- science memos
- previous guidance
- recent activity
- confidence level
- unresolved questions

The same guidance should rarely appear twice.

---

# Relationship to Other Pages

Today does not replace Story.

Today does not replace Insights.

Today does not replace Explore.

Instead:

Story collects evidence.

Insights explain patterns.

Explore expands possibilities.

Today answers:

"What should I pay attention to today?"

---

# AI Responsibilities

Today's AI acts as a coach.

Not an expert.

Not a therapist.

Not a career counselor.

Its job is to:

- recognize patterns
- encourage reflection
- recommend experiments
- celebrate progress
- ask thoughtful questions

It should never attempt to solve someone's life.

---

# Tone

Today's writing should feel:

calm

thoughtful

hopeful

specific

encouraging

It should never sound:

corporate

dramatic

motivational

therapeutic

judgmental

---

# Confidence

Confidence changes how Today speaks.

When confidence is low:

Today's guidance should ask more questions.

When confidence is high:

Today's guidance may become more specific.

Confidence changes specificity.

It never changes humility.

---

# Success

Today succeeds when users leave thinking:

"I can do that."

"I hadn't thought about it that way."

"I'm curious what happens."

The goal is not inspiration.

The goal is momentum.

---

# Anti-Patterns

Today should never:

- repeat yesterday unnecessarily
- summarize biography
- predict careers
- assign identity
- overwhelm the user
- provide multiple competing priorities
- sound like a motivational speaker

One page.

One idea.

One action.

---

# Product Rules

Today's guidance should always be:

Evidence-based.

Actionable.

Personal.

Tentative.

Current.

Small enough to complete.

Every visit should help the user move one step forward.

That is enough.
