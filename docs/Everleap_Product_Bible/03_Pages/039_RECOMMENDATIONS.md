# Recommendations

## Purpose

Recommendations transform understanding into opportunity.

They exist to help users answer one practical question:

> **"Based on everything we've learned so far, what should I explore, try, or do next?"**

Recommendations are not predictions.

They are invitations.

Every recommendation should expand a user's possibilities rather than narrow them.

---

# Philosophy

Most recommendation systems try to maximize engagement.

Everleap tries to maximize growth.

A recommendation should never exist simply because it is popular.

It should exist because it represents a thoughtful next step for this particular user at this particular point in their journey.

Recommendations should help users learn something about themselves.

Even if they decide the recommendation isn't right for them, the experience should still produce useful evidence.

---

# The Role of Recommendations

Recommendations are the bridge between self-understanding and real-world experience.

Understanding becomes meaningful only when it influences decisions.

Recommendations encourage users to:

- explore
- experiment
- practice
- build
- create
- volunteer
- join
- learn
- reflect

Every recommendation should lead to another opportunity for growth.

---

# Recommendation Types

Everleap may recommend many different kinds of experiences.

Examples include:

- Careers
- College majors
- Classes
- Clubs
- Volunteer opportunities
- Competitions
- Projects
- Books
- Podcasts
- Videos
- Mentors
- Organizations
- Certifications
- Skills to practice
- Daily habits
- Reflection exercises

The product should avoid becoming career-centric.

Life is much broader than employment.

---

# Why Recommendations Appear

Every recommendation should have a reason.

Users should understand:

- Why they are seeing it.
- What evidence supports it.
- What they might learn.
- How it connects to previous observations.

The explanation is often more valuable than the recommendation itself.

---

# Sources of Evidence

Recommendations should be informed by multiple sources whenever possible.

Examples include:

- Story responses
- Science memos
- Cross-science synthesis
- Today interactions
- Completed actions
- User interests
- Previous recommendations
- Longitudinal patterns

No recommendation should depend on a single answer.

Confidence increases as evidence accumulates.

---

# Recommendations Are Experiments

Every recommendation should be viewed as an experiment.

The product is not saying:

"This is your future."

Instead it is saying:

"This seems worth exploring."

The outcome of the experiment creates new evidence.

That evidence improves future recommendations.

---

# Personalization

Recommendations should become increasingly personal over time.

As Everleap learns more, recommendations should become:

- more specific
- better timed
- more relevant
- better explained

Users should feel the product growing alongside them.

---

# Diversity

Recommendations should balance familiarity with discovery.

Some recommendations should reinforce existing interests.

Others should introduce adjacent possibilities.

Occasionally, the product should introduce something unexpected that is still supported by evidence.

Growth often happens just outside a person's comfort zone.

---

# AI Responsibilities

The recommendation engine should:

- identify opportunities
- explain why they may fit
- connect recommendations to evidence
- encourage experimentation
- preserve user choice

It should never:

- declare a "best" path
- imply certainty
- pressure users
- overstate confidence

AI recommends.

Users decide.

---

# Measuring Success

A recommendation succeeds when it creates learning.

Not when it is accepted.

A user saying:

"I tried that and learned it wasn't for me."

is still a successful outcome.

Learning always improves understanding.

---

# Relationship to Other Pages

Story creates evidence.

Science interprets evidence.

Insights explain evidence.

Today encourages today's action.

Explore broadens possibilities.

Recommendations suggest meaningful next steps.

Every recommendation should feel like a natural continuation of the user's journey rather than an isolated suggestion.

---

# Anti-Patterns

Recommendations should never:

- rank careers
- guarantee success
- repeat generic internet advice
- recommend without explanation
- overwhelm users with options
- contradict strong evidence
- ignore changing interests

Recommendations should evolve as the user evolves.

---

# Product Rules

Every recommendation should answer five questions:

1. Why is this being recommended?
2. What evidence supports it?
3. What might the user learn?
4. Why is now the right time?
5. How will this create new evidence?

If those questions cannot be answered clearly, the recommendation should not appear.

Recommendations are not the end of the journey.

They are invitations to begin the next chapter.
